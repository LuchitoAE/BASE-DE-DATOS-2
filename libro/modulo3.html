Módulo 3: Iniciación a la Administración

3.1 SQL Server Management Studio – Administrador corporativo

SQL Server Management Studio (SSMS) es una herramienta integral para la administración de SQL Server. Permite a los administradores gestionar bases de datos, realizar consultas y configurar servidores.

3.1.1 Propiedades de la conexión actual

Las propiedades de la conexión actual se pueden visualizar en SSMS. Incluyen información como el nombre del servidor, la base de datos actual, el usuario conectado y el estado de la conexión.

Propiedad Descripción
Nombre del Servidor El nombre o dirección IP del servidor SQL Server al que está conectado.
Base de Datos Actual La base de datos en la que se está trabajando actualmente.
Usuario Conectado El nombre del usuario que ha iniciado sesión en el servidor.
Estado de Conexión Indica si la conexión está activa o inactiva.

3.1.2 Registrar un servidor SQL Server vía IP pública

Para registrar un servidor SQL Server a través de una IP pública, se deben seguir los siguientes pasos:

  1. Abre SQL Server Management Studio.
  2. Haz clic en "Conectar" y selecciona "Servidor de base de datos".
  3. En el campo "Nombre del servidor", ingresa la dirección IP pública del servidor.
  4. Selecciona el método de autenticación (Windows o SQL Server).Haz clic en "Conectar".

3.2 Creación de una BD

La creación de una base de datos en SQL Server es un proceso sencillo que se puede realizar a través de SSMS o mediante comandos SQL.

3.2.1 BD del sistema

SQL Server incluye varias bases de datos del sistema que son esenciales para su funcionamiento, tales como:

  • master: Contiene información sobre todas las demás bases de datos y la configuración del servidor.
  • model: Sirve como plantilla para nuevas bases de datos.
  • msdb: Utilizada para la programación de trabajos y la gestión de alertas.
  • tempdb: Base de datos temporal utilizada para almacenar datos temporales y objetos temporales.

3.2.2 Creación de BD propias

Para crear una base de datos propia, se puede utilizar el siguiente comando SQL:

CREATE DATABASE NombreDeLaBaseDeDatos;

También se puede crear una base de datos a través de SSMS siguiendo estos pasos:

  1. Haz clic derecho en "Bases de datos" en el Explorador de objetos.
  2. Selecciona "Nueva base de datos...".
  3. Ingresa el nombre de la base de datos y configura las opciones según sea necesario.
  4. Haz clic en "Aceptar" para crear la base de datos.

3.3 Crear un diagrama de base de datos

Los diagramas de base de datos son herramientas visuales que ayudan a representar la estructura de la base de datos. Para crear un diagrama en SSMS:

  1. Haz clic derecho en "Diagramas de base de datos" en el Explorador de objetos.
  2. Selecciona "Nuevo diagrama de base de datos...".
  3. Agrega las tablas que deseas incluir en el diagrama.
  4. Organiza las tablas y establece las relaciones entre ellas.

3.4 Adjuntar BDs

Para adjuntar una base de datos existente en SQL Server, se puede utilizar el siguiente comando SQL:

CREATE DATABASE NombreDeLaBaseDeDatos
ON (FILENAME = 'Ruta\A\Tu\BaseDeDatos.mdf'),
(FILENAME = 'Ruta\A\Tu\BaseDeDatos_log.ldf')
FOR ATTACH;

También se puede hacer a través de SSMS seleccionando "Adjuntar..." en el menú contextual de "Bases de datos".

3.5 Separar

Separar una base de datos implica desconectarla del servidor. Esto se puede hacer utilizando el siguiente comando SQL:

ALTER DATABASE NombreDeLaBaseDeDatos SET OFFLINE;
EXEC sp_detach_db 'NombreDeLaBaseDeDatos';

En SSMS, haz clic derecho en la base de datos y selecciona "Tareas" > "Desconectar" para separarla.

3.6 Copiar BD

Para copiar una base de datos, se puede utilizar el método de respaldo y restauración. Primero, realiza un respaldo de la base de datos original:

BACKUP DATABASE NombreDeLaBaseDeDatos TO DISK = 'Ruta\A\Tu\Respaldo.bak';

Luego, restaura el respaldo en una nueva base de datos:

RESTORE DATABASE NuevaBaseDeDatos FROM DISK = 'Ruta\A\Tu\Respaldo.bak';

3.7 Importar / Exportar datos

SQL Server proporciona herramientas para importar y exportar datos entre diferentes fuentes. Esto se puede hacer a través del Asistente para Importación y Exportación de SQL Server.

3.7.1 Importación de una base de Access

Para importar datos desde una base de datos de Access, selecciona "Importar datos" en el menú contextual de la base de datos de destino y sigue las instrucciones del asistente.

3.7.2 Importar datos desde Oracle

De manera similar, para importar datos desde Oracle, utiliza el Asistente para Importación y selecciona Oracle como fuente de datos.

3.8 SQL Configuration Manager

SQL Server Configuration Manager es una herramienta que permite gestionar los servicios de SQL Server, configurar las propiedades de los servidores y administrar las conexiones de red.

3.9 SQLCMD

SQLCMD es una utilidad de línea de comandos que permite ejecutar comandos T-SQL y scripts SQL desde la línea de comandos. Se puede utilizar para automatizar tareas y ejecutar scripts en SQL Server.

3.10 SQL Server Management Studio

SSMS es la herramienta principal para la administración de SQL Server. Proporciona una interfaz gráfica para gestionar bases de datos, ejecutar consultas y realizar tareas administrativas.

3.10.1 Registro de servidores

El registro de servidores permite a los administradores gestionar múltiples instancias de SQL Server desde una única interfaz. Se puede agregar un nuevo servidor haciendo clic derecho en "Conectar" y seleccionando "Servidor de base de datos".

3.10.2 Explorador de objetos

El Explorador de objetos es una herramienta en SSMS que permite navegar por las bases de datos y sus objetos, como tablas, vistas, procedimientos almacenados y más. Proporciona una vista jerárquica de todos los elementos en el servidor SQL.

3.10.3 Explorador de plantillas

El Explorador de plantillas permite a los usuarios acceder a plantillas predefinidas para crear objetos de base de datos, como tablas y procedimientos almacenados. Esto facilita la creación de objetos siguiendo un formato estándar.

3.11 Instantáneas

Las instantáneas de base de datos son copias de solo lectura de una base de datos en un momento específico. Se utilizan para proporcionar acceso a datos históricos sin afectar el rendimiento de la base de datos original.

3.11.1 Asignar nombres a instantáneas de bases de datos

Al crear una instantánea, se puede asignar un nombre descriptivo que facilite su identificación. Esto se puede hacer utilizando el siguiente comando SQL:

CREATE DATABASE NombreDeLaInstantanea AS SNAPSHOT OF NombreDeLaBaseDeDatos;

3.11.2 Conexiones de clientes con una instantánea de base de datos

Los clientes pueden conectarse a una instantánea de base de datos como lo harían con una base de datos normal. Sin embargo, es importante tener en cuenta que las instantáneas son de solo lectura y no se pueden modificar.

3.11.2.1 Restricciones a tener en cuenta sobre las instantáneas

Al trabajar con instantáneas, hay algunas restricciones que se deben considerar:

  • Las instantáneas no pueden ser utilizadas para realizar transacciones.
  • El espacio en disco utilizado por las instantáneas puede crecer rápidamente, dependiendo de la actividad en la base de datos original.
  • Las instantáneas no son una solución de respaldo; se deben realizar respaldos regulares de la base de datos original.
